# Extension Quick Start

Use this if you just want the minimum steps to get the extension working.

## 1. Start Convex

```bash
pnpm install
npx convex dev
```

## 2. Add your real config

Update `packages/extension/utils/auth.js`:

- `CLERK_PUBLISHABLE_KEY`
- `CLERK_DOMAIN`
- `CLERK_API_DOMAIN`
- Clerk cookie domains if you changed the Clerk instance

Update `packages/extension/utils/convex-client.js`:

- `CONVEX_URL`

Create or update `packages/web/.env.local`:

```bash
VITE_CLERK_PUBLISHABLE_KEY=pk_test_...
VITE_CONVEX_URL=https://your-deployment.convex.cloud
```

Set the Convex auth env var:

```bash
npx convex env set CLERK_ISSUER https://your-instance.clerk.accounts.dev
```

## 3. Configure Clerk

In Clerk:

1. Create an application
2. Enable Google sign-in if you want the current popup flow to work as-is
3. Create a JWT template for `convex`
4. Add `chrome-extension://<YOUR_EXTENSION_ID>` to allowed origins

## 4. Load the extension

1. Open `chrome://extensions`
2. Enable Developer mode
3. Click `Load unpacked`
4. Select `packages/extension`

## 5. Smoke test

1. Click `Sign in with Google`
2. Finish sign-in in the Clerk tab
3. Return to the extension and click `Sync Session`
4. Take a screenshot or recording
5. Confirm it uploads and opens in the web viewer

## If something breaks

- Check the extension console for auth errors
- Check `packages/extension/utils/auth.js` values first
- Make sure Convex types were generated by `npx convex dev`
- Make sure the extension origin is allowed in Clerk
